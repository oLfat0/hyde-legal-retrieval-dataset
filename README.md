# HyDE Legal Retrieval — Dataset (Brazilian Jurisprudence)

Dataset and experimental results for an undergraduate research project on
**Hypothetical Document Embeddings (HyDE)** applied to the retrieval of Brazilian
legal precedents.

This repository holds **only the data and results** of the project. The full
experimental pipeline (source code) lives in a separate repository.

The corpus consists of structured judicial decisions (*ementas*) from the Court
of Justice of Mato Grosso do Sul (TJMS), used to evaluate whether semantic query
expansion via large language models improves dense and hybrid retrieval.

---

## Corpus

501 public legal decisions extracted from the TJMS jurisprudence system,
focusing on civil-law rulings. Each decision is natively split by the court into
two components:

- **Cabeçalho da Ementa** (header — legal indexers and thesis): the retrieval
  **key** (the target document to be retrieved).
- **Corpo Estruturado da Ementa** (structured body — facts, controversy,
  reasoning): used as the **query**.

The structured body is further divided into sections used for the
query-granularity analysis (RQ3):

- **`curto`** — section *I. CASO EM EXAME* only (facts).
- **`medio`** — sections *I* + *II. QUESTÃO EM DISCUSSÃO* (facts + controversy).
- **`longo`** — the full structured body.

## Data Schema

### Corpus (`data/corpus/ementas_v2.json`)

A list of records, one per legal decision:

| Field | Type | Description |
|-------|------|-------------|
| `numero_processo` | string | Case number in the court system |
| `cdacordao` | string | Unique ruling ID — the relevance key linking each query to its target document |
| `classe` | string | Procedural class of the decision |
| `descricao` | string | Structured body (facts, controversy, reasoning) — used as query |
| `ementa` | string | Header (keyword-dense legal thesis) — indexed as the target document |
| `word_count` | int | Word count of the structured body |

### Queries (`data/queries/`)

- `queries_v2.json` — full-body queries for the main experiments (RQ1/RQ2).
- `queries_rq3_curto.json` / `queries_rq3_medio.json` / `queries_rq3_longo.json`
  — queries for each granularity level (RQ3).

Each query record contains at least `cdacordao` (the relevant document ID) and
`query` (the query text).

### Results (`results_*/`)

Each experiment produces a JSON file with aggregate metrics and per-query data:

| Field | Description |
|-------|-------------|
| `config` | Experiment configuration name |
| `timestamp` | Execution timestamp |
| `metrics` | Aggregate metrics (`mrr`, `recall@k`, `ndcg@k`, `n_queries`, `n_found`) |
| `per_query` | Per-query list with `rank`, `rr`, `recall@k`, `ndcg@k` |

├── results_v2/                      # main RQ1 / RQ2 results (Gemma 3 12B, N=501)
│   └── n=50/                        # pilot results on the 50-document index

│   ├── curto/
│   ├── medio/
│   └── longo/
├── results_rq3_qwen/                # RQ3 results by granularity (Qwen 2.5 14B)

> **Note on versions.** `v1` holds the first experimental design, where a generated
> mini-summary of each ruling was used as the query. This approach was later
> replaced by the `v2` design (structured body as query, header as target), which
> is the basis of the reported results. The `v1` artifacts are kept for
> reproducibility and historical comparison.
>
> **Note on models.** The main results use `gemma-3-12b-it` for HyDE generation.
> The `Qwen2.5 14B Instruct-Awq/` folder and the `results_rq3_qwen/` outputs hold
> a parallel comparison using the quantized Qwen model.

## Experimental Context

The results were produced under a controlled **auto-retrieval** protocol, where
each query has exactly one relevant document (its own header). Four retrieval
strategies were compared: dense and hybrid (dense + BM25 via Reciprocal Rank
Fusion), each with and without HyDE expansion.

- Embedding model: `facebook/mcontriever-msmarco` (768-dim).
- HyDE generation: `gemma-3-12b-it` (with a `Qwen2.5-14B-Instruct-AWQ` comparison).
- Metrics: Recall@k, MRR, nDCG@k.
